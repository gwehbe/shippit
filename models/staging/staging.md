{% docs stg_shipment_events %}
Raw source table for shipment lifecycle event data. These events are generated by different services and partners and arrive asynchronously.
{% enddocs %}

{% docs event_id %}
The unique id for a shipment lifecycle event. Prefixed with `evt_`.
{% enddocs %}

{% docs event_type %}
The shipment lifecycle event description - can be either `order_created`, `shipment_booked`, `shipment_dispatched`, `delivery_completed`, `delivery_failed`, or `delivery_returned`.
{% enddocs %}

{% docs order_id %}
The unique id for an order. Prefixed with `ord_`.
{% enddocs %}

{% docs shipment_id %}
The unique id for a shipment. Prefixed with `shp_`.
{% enddocs %}

{% docs event_timestamp %}
The timestamp of the event in local time.
{% enddocs %}

{% docs ingested_at %}
The timestamp of when the event was ingested in local time.
{% enddocs %}

{% docs carrier %}
The carrier associated with the shipment lifecycle event. Possible values are `aus_post`, `dhl`, `couriers_please`, and `aramex`.
{% enddocs %}

{% docs event_attributes %}
Additional information about the shipment lifecycle event, e.g., service level, customer account plan, etc. 
{% enddocs %}
